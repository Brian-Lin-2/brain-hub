# Architecture

A transformer is pretty straight forward. It is just a bunch of attention and feed forward layers with positional encoding.

![Transformer architecture](/docs/assets/git/transformer.png)

## Overview

Inside a transformer, it is typically just multiple layers of MHA and FFNN until an output is generated.

`Multi-Headed Attention Layer (MHA)` - Essentially parallel processing for attention. Each head will calculate their own attention and at the end the results are combined.
`Feed Forward Neural Network (FFNN)` - This is just a basic neural network. It is actually what progresses the model from the input to output text.
`Positional Encoding (PE)` - This helps assign positional weights to each token. This matters a lot for sentences as the position of words can greatly change the meaning

## Encoder

An Encoder uses `unmasked self-attention` making it bidirectional. This means every token can see every other token. This gives a form of "global context" to the Encoder. This makes Encoders good for text analysis.

## Decoder

A Decoder uses `masked self-attention` making it unidirectional. This means when generating text, a model cannot see the next words. This makes Decoders good for text generation.

## Encoder-Decoder

The traditional transformer has an encoder-decoder architecture. Basically the encoder is used to convert the sentence into a meaningful embedding and the decoder uses that embedding to generate an output. The decoder uses `cross-attention` to examine the output generated by the Encoder.

Both is needed because the Encoder helps us get the global context of the prompt. This makes sure the transformer doesn't hallucinate. However, as you know text is generated one token at a time which is where the Decoder comes in. It'll take in the embeddings and use the learned context to generate an output.
