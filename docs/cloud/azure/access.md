# Access

## Key Concepts

### Identity and Access Management (IAM)

Ensures the right people and services have the right access to cloud
resources. Access is mainly done through
`Azure Role-Based Access Control (RBAC)`.

#### RBAC

Every permission in Azure is a "Role Assignment".

> Azure RBAC is used to manage Azure Resources

- `Security Principal` - The entity requesting access
    - `User` - Individual account for a person.
    - `Group` - Collection of users managed as one unit.
    - `Service Principal`
        - Identity used by applications or automated tools to access protected resources.
        - `Client ID` - Associated ID (username)
        - `Client Secret` - Secret key for authentication (password)
        - `Tenant ID` - Represents the organization it is part of
    - `Managed Identity`
        - Identity automatically generated by Azure and tied to a specific
          resource. No key generation needed (Azure handles it). Recommended
          when possible.

#### Microsoft Entra Roles (AD Roles)

> Entra Roles are used to manage directory (Users, Groups, Domain)
